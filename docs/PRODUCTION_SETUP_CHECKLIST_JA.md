# 依頼者向け 本番環境準備・引渡しチェックリスト

- 更新日: 2026-07-16
- 目的: 依頼者が本番環境を所有し、諒さんへ本番データや秘密情報を渡さずに構築・引渡しを完了する

## 1. 実データを入れる前に依頼者が用意するもの

- 会社管理のメールアドレス
- GitHubの依頼者用アカウントまたはOrganization
- 静的Webホスティングの依頼者用Team / Project
- Supabaseの依頼者用Organization / Tokyo Project
- 本番ドメインとDNS管理アカウント
- 請求先
- 個人情報責任者と緊急連絡先

すべて依頼者名義で作成する。諒さんの個人アカウントをOwnerにしない。

## 2. アカウント作成時

- OwnerアカウントでMFAを有効にする。
- 共有IDを作らない。
- 回復コードを依頼者側の安全な保管場所へ保存する。
- 諒さんへパスワード、回復コード、API secret、DBパスワードを送らない。
- Supabaseは東京リージョンを選択する。
- 一般公開のユーザー登録を無効にする。
- クラウドのDPA、再委託先、データ処理地域、バックアップ条件を確認する。

## 3. 構築中

構築方法は次のどちらかとする。

### 方法A: 依頼者側CIで適用する

1. 諒さんが匿名データでmigration、アプリ、テストを作成する。
2. 依頼者がreleaseまたはPRを承認する。
3. 依頼者が本番secretを自分でCIへ登録する。
4. 依頼者管理CIが空の本番環境へ適用する。

この方法を優先する。諒さんは本番secretと管理画面へアクセスしない。

### 方法B: 空環境への期限付き招待

1. 実データが0件であることを確認する。
2. 諒さんを個別アカウントで期限付き招待する。
3. migrationと静的アプリを適用し、匿名データで検証する。
4. 検証後、依頼者が諒さんをGitHub、ホスティング、Supabase、DNS、監視から削除する。
5. 依頼者が全secret、DBパスワード、deploy token、hookを再発行する。
6. 削除・再発行が完了するまで実データを登録しない。

## 4. 引渡し確認

- [ ] GitHub Ownerは依頼者だけである
- [ ] Production branchへのmerge権限は依頼者だけである
- [ ] 本番deploy権限は依頼者だけである
- [ ] Supabase Owner / Adminは依頼者だけである
- [ ] 諒さんのSupabase membershipを削除した
- [ ] DBパスワードを依頼者側で再発行した
- [ ] secret key / `service_role`を依頼者側で再発行した、または未使用である
- [ ] ホスティングtoken、deploy hook、SMTP等を依頼者側で再発行した
- [ ] DNS、監視、ログ、バックアップを依頼者だけが閲覧できる
- [ ] 一般公開のユーザー登録が無効である
- [ ] 全利用者のMFAが有効である
- [ ] 未認証、MFA未完了、inactive、別organization、権限外操作の拒否テストに合格した
- [ ] 本番bundle、Git、ログにsecretと実データが0件である
- [ ] バックアップ復元を確認した
- [ ] 利用者の無効化と全session失効を依頼者だけで実行できる
- [ ] インシデント連絡網と手順を確認した
- [ ] `docs/SECURITY_REQUIREMENTS_JA.md` のP0全件を承認した

## 5. 実データ登録の開始

引渡しチェックが全件完了した後、依頼者の管理者が最初の実データを登録する。諒さんは実データ登録、画面共有、実PDF確認へ参加しない。

## 6. 納品後の保守

- 依頼者はrequest IDとエラー分類だけを共有する。
- 再現に必要な場合は、依頼者が匿名化した値または合成fixtureを作る。
- 実画面キャプチャ、実PDF、DB dump、ログ本文、バックアップを諒さんへ送らない。
- 修正は匿名環境で行い、依頼者がPRまたはreleaseを承認する。
- 本番反映は依頼者管理CIが行う。
- 緊急時でも常設の開発者用管理者アカウントを作らない。

## 7. 依頼者が最終決定する項目

- 初期利用者と各役割
- 利用端末が会社管理端末かBYODか
- 作業員、退職者、現場、監査ログ、バックアップの保持期間
- PDFの提出先、保管場所、削除時期
- 個人情報責任者、事故時連絡先
- 定期的な権限棚卸し、復元試験、脆弱性診断の担当者

これらはチャットで秘密情報や実データを共有せず、依頼者の社内記録として管理する。
